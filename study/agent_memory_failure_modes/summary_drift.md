# 失败模式一：摘要漂移（Summary Drift）

摘要漂移是长期 memory 系统里最常见、也最隐蔽的失败模式之一。  
它指的是：系统原本写入的是合理摘要，但随着多轮整合、重写、压缩和再摘要，记忆内容逐步偏离原始证据，最后形成一个“看起来很顺、其实已经失真”的长期记忆对象。

它危险的地方就在这里：  
漂移后的 memory 往往不像原始 hallucination 那样明显离谱，反而会显得更简洁、更稳定、更像“已经被系统确认过的事实”。

## 一、摘要漂移通常怎么发生

最常见的链路是这样的：

1. 原始消息本来是细致而有条件的。
2. 第一次摘要时，保留了大部分意思，但丢掉一点限定条件。
3. 后续系统不是回原文，而是基于“上一次摘要”再继续摘要。
4. 几轮以后，系统保留下来的只剩一个高度压缩、条件缺失的版本。

例如原始交互是：

> “最近可以少量喝牛奶了，但点咖啡还是优先燕麦奶。”

第一次摘要可能写成：

> “用户对牛奶耐受有所恢复，咖啡默认燕麦奶。”

再往后，如果系统继续压缩，可能变成：

> “用户可以喝牛奶，偏好燕麦奶。”

再糟一点，最后可能只剩：

> “用户可以喝牛奶。”

这就是典型的 summary drift：  
不是一句话突然变错，而是每一轮都“损失一点点”，最后偏离原意。

## 二、为什么摘要漂移这么危险

第一，它会伪装成“系统更聪明了”。  
因为摘要通常比原文更短、更像规范化知识，结果使用者反而更容易信它。

第二，它会跨阶段污染系统。  
一旦错误摘要写进长期 memory，后续检索、规划、回答都会在这个错误对象上继续构建。

第三，它不容易被简单 benchmark 发现。  
如果 benchmark 只看最终回答是否大致相关，而不追溯中间 memory object，漂移很可能长期潜伏。

## 三、最容易发生漂移的几类场景

### 1. 带例外条件的偏好

如：

- “平时……但特殊情况下……”
- “默认……但不是总是……”
- “最近有变化，但主偏好没变……”

这类信息特别容易在摘要中丢掉转折和优先级。

### 2. 时间敏感事实

例如：

- “之前是 A，现在改成 B”
- “这周如此，下周可能调整”

如果摘要不显式建模时间，系统很容易把变化压平。

### 3. 多来源合并

如果同一 memory object 来自多轮、多 session、多来源整合，漂移风险会更高。  
因为系统往往是在摘要上继续摘要，而不是每次都回原始证据。

## 四、哪些方法特别容易踩这类坑

纯 summary-heavy 的方法更容易出现摘要漂移，比如：

- `MemoChat`
- 某些 aggressive compaction 变体
- 过度依赖上轮摘要做下一轮聚合的系统

`A-MEM`、`MemTree` 这类也有这个风险，只是它们通常会通过链接或层级结构缓解一部分。  
`Mem0` 风格系统如果保留 raw fallback，会更稳；  
`MemoryOS` 这类分层系统如果不强制长期层只从稳定证据晋升，也一样会漂。

## 五、怎么发现摘要漂移

最有效的不是看最终答案，而是看中间痕迹。

### 1. 对比原始证据和当前 memory object

问自己：

- 原文里的限定词还在吗？
- 时间变化还在吗？
- 例外条件是否被保住？
- 结论语气是不是被错误加强了？

### 2. 看摘要是不是在“基于摘要继续摘要”

如果一个系统的 consolidation pipeline 反复以旧摘要为输入，而很少回源原文，它几乎一定存在 drift 风险。

### 3. 建立“source-backed memory audit”

每条重要摘要最好都能追溯：

- 来源会话
- 来源消息
- 摘要生成时间
- 最近一次更新时间

没有这些元信息，漂移几乎无从 debug。

## 六、工程上怎么缓解

### 1. 永远保留 raw evidence

这是最基本的。  
摘要应该是派生层，而不是唯一层。

### 2. 重要摘要不要只基于旧摘要更新

更新时要尽量回到：

- 原始消息
- 最近几条关键证据
- 或者至少是带 provenance 的片段集合

### 3. 对“高风险句式”做特殊保护

例如：

- “但”
- “除非”
- “最近”
- “暂时”
- “默认”
- “优先”

这些词往往决定语义边界，摘要时不应轻易丢掉。

### 4. 给 memory object 加置信等级和稳定等级

把“暂时观察”与“长期稳定事实”分开，不要一写进 memory 就自动当作稳定知识。

## 七、如果自己做系统，这里最值得加的机制

我最推荐的两个机制是：

1. **summary-to-source backpointer**
   每个摘要都能跳回来源原文。

2. **periodic source-grounded refresh**
   定期不是基于旧摘要，而是基于原始证据重新生成关键摘要。

这两个机制会大幅降低“摘要雪球越滚越偏”的风险。

## 八、最后一句总结

摘要漂移最可怕的地方不是它会错，而是它会**错得越来越像真理**。  
所以只要系统里有长期摘要层，就必须默认这个风险长期存在，并为它留出回源与审计路径。
